margin, border and padding

Margin, border and padding are all part of the box model of CSS. The box model is a way of thinking about the design and layout of a page, breaking down each individual element into boxes. Any HTML element can be considered a box, and we can use these boxes in CSS to change the design and layout of our page. HTML elements include things like: title, heading, image, paragraph etc. We will be using the analogy of a framed picture on a wall to explore this concept.

The border is the edge of the element. If you think about a framed picture, the border is like the wooden frame itself. Borders by default are ‘invisible’, not having any size or width, but this can be changed with CSS. Even if the borders remain ‘invisible’, margin and padding still apply. See the example below:

The margin is the space outside the edge of the element. If you think about a frame hung on a wall, the margin is the space between the edge of the frame and the wall. Just like how we can move a frame to sit in a certain position on the wall, we can control the margin of an element from all sides (top, bottom, left, right). See the example below:

This paragraph has a blue border 2 pixels wide and margins of 200px on each side in relation to the transparent box.

Margins are used when spacing elements in relation to each other, just like how you might space two frames apart on a wall. Margins can be used to space elements on a page, up or down and left and right. You can use margins to centre horizontally or vertically. You can also use margins to put more space between elements on the same page.

Padding is the space inside an element, between its content (ie. words or images) and the border. Imagine padding is like the matte board inside a frame, it puts space between the frame and the picture itself. Just like margins, padding can be controlled from all sides (top, bottom, left, right).
